## Content Guidelines

**Voice:**
- Builder-grade, calm, precise
- No buzzwords ("transformation," "innovative")
- Evidence over adjectives
- Numbers and deltas, not feelings

**Structure:**
- H2 sections (scannable)
- Code examples when relevant
- Metrics and numbers (quantified)
- Real screenshots/diagrams when available

**Length:**
- Case studies: 2,000-3,000 words
- Concept posts: 1,000-1,500 words
- Technical deep-dives: 1,500-2,500 words

**SEO:**
- Meta description (150-160 chars)
- H1 = title, H2 = sections
- Internal links (2+ to other pages)
- Keywords natural, not stuffed
